# AI Lab — Intelligent Content System

AI Lab is a structured system for building, testing, and evolving AI-driven content workflows.

It is not a blog.
It is not documentation.

It is a content operating system powered by Codex executions.

---

## System Overview

AI Lab has four linked layers:

`Codex -> SEO Engine -> Blog -> Experiments`

- Codex: execution layer (how systems run)
- SEO Engine: visibility layer (how systems are discovered)
- Blog: output layer (what is published)
- Experiments: exploration layer (what is tested)

---

## Core Philosophy

1. Content is executed, not written.
2. Everything must be traceable.
3. Structure is more important than isolated posts.
4. The system evolves through execution.

---

## Core Components

### Codex (Execution Core)

Manages prompts, workflows, executions, and integrations.

Entry: `/codex`

### SEO Engine (Visibility Layer)

Manages keyword clusters, content graph, and ranking experiments.

Entry: `/seo-engine` (planned)

### Blog (Output Layer)

Publishes executed outputs (articles, case studies, reports).

Entry: `/blog` (planned)

### AI Experiments (Exploration Layer)

Runs unstable or early-stage tests.

Entry: `/ai-experiments` (planned)

---

## Content Lifecycle

`Prompt -> Codex Workflow -> Execution Run -> SEO Index -> Blog Output`

---

## Linking Rules

- Blog post -> codex workflow + execution
- Execution -> commit hash (if available)
- SEO page -> related workflow + posts
- Experiment -> prompt or workflow id

---

## Why AI Lab Exists

Traditional blogs answer: "what was written?"

AI Lab answers: "how was it generated, and can it be reproduced?"
